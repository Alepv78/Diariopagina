1.-Proceso de creacion del sitio web

El proceso de creacion del sitio web a sido un largo camino de aprendizaje ya que comenzamos aprendiedo la estructura HTML y posterior a ello fuimos empezando a ver las etiquetas mas básicas de HTML hasta llegar a las etiquetas as avanzadas que hoy estamos usando en nuestro trabajo final que es una pagina referente a las 7r donde podemos observar desde etiquetas básicas como el titulo hasta etiquetas mas complejas como el añadir videos tablas, imágenes y hasta audios, y sobre todo los tipos de fuentes y color de fondo para darle una vista mas bonita a todo y sea mas armonioso.


2.-Dificultades encontradas

Durante nuestro trabajo en el curso hubo una serie de dificultades que nos afectan de manera considerable entre las mas destacables encontramos el uso incorrecto de las etiquetas, por ejemplo en algunas ocasiones escribiamos las etiquetas con la sintaxis incorrecta y como consecuencia no nos aparecia reflejado en nuestra pagina y teniamos que ver nuestro error en el codigo y aveces es dificil encontraro.

Otra de nuestras dificultades en que algunos alumnos no hacian su respectivas tareas asiganadas por el docente y por consiguiente hay mucha copia entre paginas de compañeros y no aprenden nada.


3.-Soluciones

Despues de analizar nuestras dificultades es momento de proponer soluciones que nos ayuden con nuestros problemas, en mi opinion yo propongo que para no cometer errores al momento de usar las etiquetas mas complejas, sea necesario tenerlas con un ejemplos en nuestro cuaderno para que veamos la sintaxis correcta sin miedo a equivocarnos.

Otra solucion interesante que yo propondria es que los alumnos que entienden a la perfeccion el uso de ls etiquetas tengan un rol de ayudar a aquellos compañeros que no son tan experimentados en la materia y asi reciban el apoyo de los compañeros mas inteligentes del salon y puedan entender mejr lo que hace y resolver dudas.

Y por ultimo propongo que para evitar que haya muchas copias entre los alumnos el maestro lleve un control mas especifico de las actividades y que les asine un dia de entrega obligatorio para todos y asi los alumnos entan obligados ahcer sus trabajos en la fecha asiganada sin tener que copiarselo a otro.

4.-Reflexiones

Crear una página web en HTML es un viaje que, aunque parece sencillo al principio, está lleno de pequeños desafíos y lecciones valiosas. Al principio, todo es nuevo: etiquetas, estructuras, y el concepto de cómo se visualiza algo en un navegador.Aveces nos sentimos un poco perdidos, tal vez mirando líneas de código que no entiende del todo, o intentando que las imágenes se alineen correctamente, pero al no encontrar la solución de inmediato, la frustración puede empezar a surgir. Es un proceso que parece estar lleno de obstáculos: errores en el código que no logran identificarse, características de diseño que no se ven como deberían, y dudas acerca de si el proyecto realmente va a salir bien.

Pero, a medida que sigue trabajando, algo empieza a cambiar. Nos damos cuenta que empezamos a entender cómo las etiquetas se relacionan entre sí, cómo una estructura básica de HTML puede evolucionar para incluir más interactividad y diseño con el tiempo. La clave está en la perseverancia. Cada error es una oportunidad para aprender algo nuevo. En lugar de desanimarse, aprende a buscar soluciones, ya sea a través de tutoriales, foros, o simplemente experimentando con diferentes enfoques.

Lo interesante de este proceso es que, cuando finalmente el vemos nuestra página funcionando correctamente con todos los elementos alineados, el contenido bien organizado, y el diseño funcionando como debería—hay una satisfacción profunda. Ese sentimiento de haber superado los obstáculos, de haber construido algo tangible desde cero, es una recompensa enorme.


5.-Mejoras propuestas

Organizar mejor tu código: Mantén una estructura clara de carpetas (HTML, CSS, imágenes, scripts) para facilitar la navegación y el crecimiento del proyecto. Comenta tu código: Añadir comentarios ayuda a entender la función de cada bloque, facilitando el mantenimiento y futuras modificaciones. Usa plantillas o frameworks: Comienza con una base como Bootstrap para acelerar el desarrollo y enfocarte en contenido y funcionalidad. Aprovecha las herramientas del navegador: Las DevTools permiten depurar y ajustar el código en tiempo real, mejorando la eficiencia. Valida tu código: Utiliza validadores como W3C para asegurar que el código cumpla con los estándares y funcione bien en diferentes navegadores. Consulta la documentación: Leer la documentación oficial de HTML, CSS y JavaScript te dará una comprensión más profunda del código. Diseño responsivo: Implementa un diseño que se ajuste a diferentes tamaños de pantalla desde el inicio, asegurando una buena experiencia en todos los dispositivos. Prueba en varios navegadores: Asegúrate de que tu página funcione correctamente en diferentes navegadores para evitar problemas de compatibilidad. Planifica antes de programar: Crear un esquema o wireframe antes de escribir código facilita la toma de decisiones y evita rehacer partes del proyecto. Colabora y recibe feedback: Compartir tu trabajo con otros y recibir retroalimentación puede mejorar significativamente la calidad de tu proyecto. Usa control de versiones (Git): Git permite seguir el historial de cambios, colaborar eficientemente y mantener el código organizado.
